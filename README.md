# ECサイトにおける消費者行動分析と商品推薦

## 概要

本プロジェクトでは、ECサイトのユーザー行動データを用いて、

- 消費者がどのような商品に興味を持つのか
- どのような条件で購買に至るのか
- どのような商品が売れやすいのか

を分析する。

最終的には、分析結果をもとに商品推薦やECサイト改善への提案を行うことを目的とする。

---

# 分析目的

ECサイト上におけるユーザー行動を分析し、消費者の興味・購買傾向を把握する。

特に、

- 閲覧(view)
- ウィッシュリスト追加(wishlist)
- カート追加(cart)
- 購入(purchase)
- レビュー(review)

などの行動から、ユーザーが購入に至るまでの流れを整理し、商品の特徴やユーザー行動との関係性を分析する。

---

# 使用データ

使用データセット：

- Kaggle : E-commerce_dataset

主に以下のようなデータを利用する予定。

| データ | 内容 |
|---|---|
| users | ユーザー情報 |
| products | 商品情報 |
| orders | 注文情報 |
| order_items | 購入商品情報 |
| events | ユーザー行動ログ |

---

# 分析方針

## 現時点で想定している特徴量

| 特徴量 | 想定する意味 |
|---|---|
| 購買数 | 商品需要・人気 |
| レビュー評価 | 商品満足度 |
| 価格 | 購買ハードル |
| ウィッシュリスト追加数 | 比較・検討段階 |
| 閲覧数(view) | 商品への興味 |
| カート追加(cart) | 購買意欲 |

---

# 仮説

現段階では、

> 「購買数とレビュー評価がともに高い商品は人気商品である」

という仮説を置いて分析を進める。

まずは上位商品の傾向を確認し、

- カテゴリ
- 価格帯
- レビュー傾向
- 行動データ

などとの関係性を分析する。

---

# 分析ロードマップ

## Phase1 : 人気商品の把握

以下を確認する。

- 購買数上位の商品
- レビュー評価の高い商品
- 購買数と評価の両方が高い商品
- 上位商品の特徴分析

---

## Phase2 : 消費者行動分析

ユーザー行動を以下のように整理する。

| 行動 | 意味 |
|---|---|
| view | 興味 |
| wishlist | 比較・検討 |
| cart | 購買意欲 |
| purchase | 実購入 |
| review | 満足度 |

これらをもとに、

- 興味は高いが購入されない商品
- 購入率の高い商品
- カテゴリごとの行動傾向

などを分析する。

---

## Phase3 : 売れやすい商品の条件分析

人気商品や購入率の高い商品の特徴を整理し、

- 価格帯
- レビュー傾向
- 商品カテゴリ
- 行動傾向

などから、売れやすい商品の条件を分析する。

---

## Phase4 : 商品推薦・改善提案

分析結果をもとに、

- 商品推薦
- 類似商品の提案
- 人気商品の特徴整理
- ECサイト改善提案

などにつなげる。
